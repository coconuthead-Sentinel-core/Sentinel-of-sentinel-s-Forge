## How to Use This Log

Record **every** information security incident, data breach, accessibility failure, or nonconformity here.

**When to log:**
- Any unauthorized access to credentials or data
- Any AI output that stigmatizes, demeans, or discriminates against neurodivergent cognition
- Any system failure affecting availability or accessibility
- Any GDPR data breach — especially involving neurodivergent health data (Art. 9)
- Any accessibility failure that makes the platform unusable for neurodivergent users
- Any CI/CD pipeline failure caused by security issues

## Data Breach Notification Checklist (GDPR Articles 33 & 34)

If a breach involves personal data — especially neurodivergent health/disability data:

**Within 72 hours of detection:**
- [ ] Incident contained
- [ ] Nature and scope documented
- [ ] Categories of data involved (standard personal / special category / Art. 9 health)
- [ ] Estimated number of individuals affected
- [ ] Likely consequences assessed
- [ ] Measures taken to address breach documented
- [ ] **Notification to supervisory authority filed** (if required)

**If high risk to individuals (especially neurodivergent users):**
- [ ] **Notify affected individuals without undue delay**
- [ ] Communication written in accessible format (ADHD/dyslexia-friendly)
