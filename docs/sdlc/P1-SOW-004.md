## 2. Executive Summary

This SOW covers the engineering development and production hardening of Sentinel Forge, an enterprise-grade cognitive AI orchestration platform. The work includes: building a FastAPI backend with 76 API routes, implementing JWT authentication and 4-tier RBAC, integrating Stripe billing with 3 subscription tiers, adding Azure Cosmos DB with migration system, deploying via Docker Compose with nginx TLS proxy, writing 40 automated tests, and producing complete SDLC documentation.

- **Business objective**: Deliver a production-ready AI platform that can generate subscription revenue
- **Primary users/stakeholders**: Platform developers, enterprise teams, platform administrators
- **Definition of success**: 40/40 tests passing, 76 routes functional, clean startup, all security systems operational
- **In-scope platforms**: Web API (REST + WebSocket), HTML/JS frontend dashboard

## 3. Scope of Work

### In-scope capabilities (functional):
- Cognitive processing pipeline with 4 processing modes
- Symbolic rule engine (CRUD + interpretation)
- Memory management (snapshots, active nodes, entropy)
- Multi-provider AI adapters (Azure OpenAI, Claude, ChatGPT, Gemini) with mock fallback
- JWT authentication (signup, login, refresh, profile)
- 4-tier RBAC (Viewer/User/Operator/Admin)
- Stripe billing (3 plans: $29/$99/$499, checkout, portal, webhooks)
- WebSocket real-time streaming (sync, metrics, events)
- Event bus with overflow policies (drop, latest, block, error)
- Database migration system (4 migrations, upgrade-on-read)
- Pool management (create, teardown, rebuild, stress test)
- Notes management (upsert, list)
- Social features (friendships, guilds)
- Dashboard endpoints (metrics, activity, sentinel view)
- Health checks (/healthz, /readyz)
- Prometheus metrics exposition (/metrics/prom)
- Ops dashboard (HTML)

### In-scope integrations:
- Azure Cosmos DB (NoSQL, with mock fallback)
- Azure OpenAI (chat completions, embeddings)
- Azure Identity (AAD token authentication)
- Stripe (checkout sessions, subscriptions, webhooks, billing portal)

### In-scope data:
- Notes (text, tag, vector, metadata)
- Memory snapshots (summary, active_nodes, entropy_level)
- User records (email, hashed password, role, subscription tier)
- Events (JSON, streamed via SSE and WebSocket)

### In-scope platforms/environments:
- Development (MOCK_AI=true, in-memory storage)
- Production (Azure Cosmos DB, Azure OpenAI, Stripe live keys)
- Docker Compose deployment with nginx proxy

### Non-functional requirements:
- TLS 1.2/1.3 (nginx with modern cipher suites)
- Request size limit: 10MB
- Rate limiting: 600 RPM / 120 burst (configurable)
- Structured JSON logging (ELK/CloudWatch/Datadog compatible)
- Health checks for container orchestration

### 4.6 Security, Privacy, and Compliance

- **Authentication**: JWT (HS256) with bcrypt password hashing; API key authentication
- **Authorization model**: RBAC with 4 tiers (Viewer < User < Operator < Admin)
- **Data protection**: TLS 1.2/1.3 in transit (nginx); bcrypt at rest (passwords); secrets via environment variables
- **PII handling**: User email and hashed password stored; no plaintext passwords; JWT errors sanitized
- **Audit logging**: Structured JSON logging with request_id, user_id, endpoint, duration_ms
- **Threats & mitigations**: Request size limit (10MB), rate limiting (600 RPM), CORS configuration, security headers (HSTS, CSP, X-Frame-Options)
- **Security reviews required**: Recommended before production launch (not yet performed)
